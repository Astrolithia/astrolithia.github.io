静态工厂模式
系统支持多种代码沙箱：

  • ExampleCodeSandbox（示例沙箱，用于测试）
  • RemoteCodeSandbox（远程沙箱，调用独立部署的沙箱服务）
  • ThirdPartyCodeSandbox（第三方沙箱）
    如果用 if-else 来切换

通过在 Spring 配置，一行配置切换沙箱

代理模式

每种沙箱调用前后都需要打印日志：

用静态代理，创建一个代理类，专门负责增强逻辑，让原有实现类保持干净：

好处： - 原有实现类完全不需要改动 - 日志逻辑只写一次，不重复 - 符合单一职责原则：实现类只负责核心逻辑，代理类负责增强

第四步：判题服务调用

Java 安全管理器
Java 提供的安全性工具，用于控制 Java 应用程序对系统资源的访问和执行权限。它通过定义安全策略文件来管理和限制 Java 程序的操作，以确保程序不会越权访问系统资源或执行危险操作
在项目中，我通过继承 SecurityManager 类自定义了一个安全管理器，通过覆写 checkWrite 和 checkExec 的方式限制了禁止用户写文件和执行文件 然后修改代码沙箱执行 Java 代码的命令，绑定定义好的安全管理器，从而限制了用户代码中的部分危险操作
