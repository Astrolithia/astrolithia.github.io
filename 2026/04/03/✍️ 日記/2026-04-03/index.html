Java 中的变量

局部变量
在方法体内声明的变量，该变量只能在该方法内使用，类中的其他方法并不知道该变量

访问修饰符不能用在局部变量
在栈上分配
没有默认值，被声明后必须初始化才能使用

成员变量
在类内部但在方法体外声明的变量称为成员变量或者实例变量，只能通过类的实例来访问

至少被一个方法、构造方法或者语句块引用
可以声明在使用前或使用后
访问修饰符可以修饰成员变量

静态变量
static 声明的变量被称为静态变量

必须在方法构造方法和语句块之外
除了被声明为常量意外很少用
可以在静态语句块中初始化

常量
final 修饰的成员变量称为常量

Java 中的方法

实现代码的可重用性，编写一次方法，可以多次使用

四种访问权限修饰符：

  1. public：可以被所有类访问
  2. private：只能在定义它的类中访问
  3. protected：可以被同一个包中的类，或者不同包中的子类访问
  4. default：只能被同一个包中的类访问

方法名，最好用驼峰式

方法分为标准类库方法和用户自定义方法

预先定义好的方法称为标准类库方法
自定义的方法称为用户自定义方法

没有使用 static 关键字修饰，但在类中声明的方法被称为实例方法，在调用实例方法之前，必须创建类的对象

用 static 修饰的方法被称为静态方法

没有方法体的方法被称为抽象方法，总是在抽象类中被声明，用 abstract 修饰

Java native 方法

JNI 一开始是为了本地已编译语言，尤其是 C 和 C++而设计拆，使用 Java 与本地已编译的代码交互，会丧失平台可移植性，但是有些情况下这样做是必须的
JNI 的缺点：

程序不再跨平台，想要跨平台必须在不同的系统环境下重新编译本地语言部分
程序不再绝对安全

用 C 语言编写程序本地的方法

  1. 编写带有 native 方法的 Java 类，生成 .java 文件
  2. 使用 javac 命令编译所编写的 Java 类，生成 .class 文件
  3. 使用 javah -jni java 类名 生成扩展名为 h 的头文件，也即生成 .h 文件
  4. 使用 C/C++或者其他编程语言，创建 .h 文件的实现，也就是创建 .cpp 文件实现 .h 文件中的方法
  5. 将 C/C++编写的文件生成动态连接库，生成 ddl 文件

抽象类

抽象类不能通过 new 关键字实例化，但是可以有子类，子类通过 extends 关键字来继承抽象类 至少有一个抽象方法 子类必须给出父类中的抽象方法的具体实现 除非该子类也是抽象类

使用场景：

希望一些通用的功能被多个子类复用的时候
需要在抽象类中定义好 API，然后在子类中扩展实现的时候可以使用抽象类

Java 接口

接口通过 interface 关键字来定义，可以包含一些常量和方法
接口中定义的所有变量或者方法 在编译的时候都会自动添加上 public static final 修饰符

没有使用 private、default 或者 static 关键字修饰的方法是隐式抽象的
从 Java 8 开始 接口中允许有静态方法 但是静态方法无法由类的对象调用 只能通过接口名调用
接口中的默认方法 为实现该接口而不覆盖该方法的类提供默认实现

接口有 default 方法也是由 Java 8 开始的 如果接口新增了一个抽象方法 所有实现类都必须实现它 改动量很大 而用 default 方法就可以只改接口一处

接口不允许实例化
接口可以是空的 最典型的例子就是 Serializable 接口
不要再定义接口的时候使用 final 关键字
接口的抽象方法不能是 private、protected 或者 final，不然会报错

接口的作用

使某些实现类具有我们想要的功能 比如实现 Cloneable 接口的类具有拷贝的功能

Java 原则上只支持单一继承 但是通过接口可以实现多重继承的目的

实现多态
多态就是同一个事件发生在不同的对象上会产生不同的结果 比如鼠标点击左上角的 x 可以关闭窗口 点击超链接可以打开网站
多态存在的三个前提：
要有继承关系 两个类都实现了同一个接口
子类要重写父类的方法
父类引用指向子类对象

策略模式

策略模式的思想是 针对一组算法 将每一种算法封装到具有共同接口的实现类中 接口的设计者可以在不影响调用者的情况下对算法做出改变

适配器模式

针对调用者的需求对原有的接口进行转接

工厂模式

抽象类和接口的区别

通过关键字 abstract 定义的类叫做抽象类

接口在 Java 中是一个抽象类型，是抽象方法的集合，通过 interface 来定义
接口是隐式抽象的 所以声明时没有必要使用 abstract 关键字
接口的每个方法都是隐式抽象的，所以同样不需要使用 abstract 关键字
接口中的方法都是隐式 public 的

抽象类可以包含具体方法的实现 而在接口中 方法默认是 public abstract 的，但从 Java 8 开始，接口也可以包含有实现的默认方法和静态方法

Java static 关键字的作用

静态变量

如果在声明变量的时候使用 static 关键字，那么这个变量就被称作静态变量。静态变量只在类加载的时候获取一次内存空间，这使得静态变量很节省内存空间

假设郑州大学录取了一万名新生，那么在创建一万个 Student 对象的时候，所有的字段（name、age 和 school）都会获取到一块内存。学生的姓名和年纪不尽相同，但都属于郑州大学，如果每创建一个对象，school 这个字段都要占用一块内存的话，就很浪费

静态方法

静态方法不能访问非静态变量和调用非静态方法

静态代码块

静态代码块通常用来初始化一些静态变量 它会先于 main() 方法执行

在 Java 1.6 可以不存在 main 方法也能执行成功 Java 7 之后就没办法了
静态代码块在初始化集合的时候 非常有用
在实际的项目开发中 通常使用静态代码块来加载配置文件到内存当中

静态内部类

不太常用

第一次加载 Singleton 类时并不会初始化 instance，只有第一次调用 getInstance() 方法时 Java 虚拟机才开始加载 SingletonHolder 并初始化 instance，这样不仅能确保线程安全，也能保证 Singleton 类的唯一性

Final 关键字

被 final 修饰的变量无法重新赋值

final 修饰的成员变量必须有一个默认值 否则编译器会提醒没有初始化
final 和 static 一起修饰的成员变量叫做常量 常量名必须全部大写

final 类

String 类设计成 final 的原因

  1. 为了实现字符串常量池
  2. 为了线程安全
  3. 为了 HashCode 的不可变性

如果一个类只是 final 的，那么它并不是不可变的全部条件
把一个类设计为 final 的，有其安全方面的考虑，但不应该故意为之

不可变对象

什么是不可变类

一个类的对象在通过构造方法创建后如果状态不会再改变，那么它就是一个不可变类 所有成员变量的赋值仅在构造方法中完成

常见的不可变类

  1. 常量池的需要
  2. hashCode 需要
  3. 线程安全

手撸一个不可变类

必须要满足以下 4 个条件

  1. 确保类是 final 的，不允许被其他类继承
  2. 确保所有的成员变量（字段）是 final 的，这样的话，它们就只能在构造方法中初始化值，并且不会再随后被修改
  3. 不要提供任何 setter 方法
  4. 如果要修改类的状态，必须返回一个新的对象

不可变类有很多优点，尤其是在多线程环境下，它非常的安全，尽管每次修改都会创造一个新的对象，增加内存的消耗

方法重写和方法重载

  • 重写的方法必须和父类中的方法有着相同的名字；
  • 重写的方法必须和父类中的方法有着相同的参数；
  • 必须是 is-a 的关系（继承关系）。
  • 重写的方法必须返回相同的类型。
  • 重写的方法不能使用限制等级更严格的权限修饰符。
  • 可以在子类中通过 super 关键字来调用父类中被重写的方法
  • 构造方法不能被重写
  • 如果一个类继承了抽象类，抽象类中的抽象方法必须在子类中被重写
  • synchronized 关键字对重写规则没有任何影响
  • strictfp 关键字对重写规则没有任何影响

Java 注解

注解的生命周期有三种策略

  1. SOURCE：在源文件中有效，被编译器丢弃
  2. CLASS：在编译器生成的字节码文件中有效，但在运行时会被处理类文件的 JVM 丢弃
  3. RUNTIME：在运行时有效。这也是注解生命周期中最常用的一种策略，它允许程序通过反射的方式访问注解，并根据注解的定义执行相应的代码

EnumMap 的效率比 HashMap 还要高，可以直接通过数组下标（枚举的 ordinal 值）访问到元素

单例用来保证一个类仅有一个对象 并提供一个访问它的全局访问点 在一个进程中 因为这个类只有一个对象 所以就不能再使用 new 关键字来创建新的对象
