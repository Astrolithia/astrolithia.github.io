Tui CLI GUI
【10分钟带你从0配置最强命令行】 https://www.bilibili.com/video/BV1fdTfzeE8X/ 模糊搜索
如果你把“高级工程师用 Vim”想象成“会很多快捷键”，那其实还只是表层。真正的高手，是把 Vim 当成一个“思维放大器 + 自动化引擎”。

我给你讲一个“Aric的一天”，你会更直观👇

☀️ 09:30 — 打开项目（不是打开文件，是进入“战场”）

Aric 不会慢慢点文件夹，他直接：

  • vim . + 配合 fzf
  • 几个字符模糊搜索 → 秒开文件

👉 他不是“找文件”，而是“跳到目标”

🔍 10:00 — 阅读代码（像在“扫雷图”上移动）

普通人：用鼠标滚

Aric：

  • /UserService → 精准搜索
  • n / N → 上下跳
  • gd → 跳到定义
  • Ctrl-o → 返回

👉 他的大脑在“路径跳转”，不是“阅读滚动”

✏️ 10:30 — 修改代码（操作的是“结构”，不是字符）

看到一段代码：

if (user != null) {

    user.setName(“test”);

}

Aric 不会一个个删改，他直接：

  • ci( → 改括号内
  • di{ → 删除代码块
  • :%s/user/customer/g → 全局替换

👉 Vim高手操作的是“语义块”，不是键盘输入

⚡ 11:30 — 批量操作（人肉脚本 vs 真脚本）

需求：修改 50 个文件里的某个字段

普通人：一个个改

Aric：

  • :args **/*.java
  • :argdo %s/old/new/g | update

👉 一条命令，搞定整个项目

🧠 14:00 — 思考复杂逻辑（Vim进入“IDE模式”）

他用的不是裸 Vim，而是：

  • Neovim + LSP
  • 自动补全
  • 跳转引用
  • 错误提示

但重点是：

👉 他仍然“键盘驱动一切”，没有鼠标打断思路

🔁 15:30 — 重复操作（录制宏）

场景：重复修改10行结构类似代码

Aric：

  • qa → 开始录制
  • 做一遍操作
  • q → 结束
  • @a → 重复执行
  • 10@a → 批量执行

👉 Vim = 可编程编辑器

🧩 17:00 — 和终端融合（不是工具切换，是“一个世界”）

他不会：

  • 切到终端 → 再回来

他直接在 Vim 里：

  • :terminal
  • 或配合 tmux

👉 编辑 + 编译 + 运行 = 一体化

🌙 21:00 — 收尾（不是保存文件，是管理状态）

Aric：

  • :wa → 保存全部
  • :mksession → 保存工作现场
  • 第二天 vim -S → 恢复一切

👉 Vim = 工作流操作系统

🧠 本质总结（这才是重点）

高级工程师用 Vim，不是因为“酷”，而是因为：

1️⃣ 操作的是“文本结构”，不是字符

  • 单词 / 句子 / 代码块 / 函数

2️⃣ 一切都可复用

  • 宏
  • 命令
  • 脚本

3️⃣ 减少“上下文切换”

  • 不用鼠标
  • 不跳工具

🔥 给你一个现实建议（很重要）

你现在的阶段，不要学“全部 Vim”，只学这 5 个就够：

  • ci（change inside）
  • di（delete inside）
  • /（搜索）
  • n/N（跳转）
  • .（重复上一步）

👉 这 5 个，就能秒杀 90% 初级用户
