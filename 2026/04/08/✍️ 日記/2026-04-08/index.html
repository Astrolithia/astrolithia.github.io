🧠 学到了什么（知识/成长）

AI 编程对话技巧

  • 需求要具体，一次改 1~5 个点
  • 遇到问题直接说”XX 有问题”
  • 不懂的代码可以要求 AI 解释

各 AI 模型适用场景

  • 大项目/高代码质量 → Claude + Claude 4.7 Opus
  • 快速原型 / 验证想法 → Claude + Gemini（速度快、成本低）
  • VS Code 阅读/开发 → Claude + DeepSeek
  • 追求速度 → GLM（编程强、中文理解好、API 便宜）
  • 大量测试 → DeepSeek / 通义千问
  • 快速验证想法（零代码）→ bolt.new 等平台
  • 生成图片 → Nano（中文支持好）或 Midjourney
  • 生成视频 → Kimi AI 或 Veo 3（高质量但贵）
  • 日常问答 → 豆包或元宝
  • 图片处理/去水印 → 百度文心助手
  • 分析大型项目/长上下文 → Gemini
  • 调研国外技术 → Gemini 网页版（联网搜索+深度分析）

AI 多媒体创作工具

  • Nano：中文支持好，图片风格统一，适合教程配图
  • 即梦 AI（字节跳动）：文字生图、图生视频，适合生成视频空镜画面
  • Veo 3（谷歌）：最先进的 AI 视频生成，物理光影真实，但价格昂贵
  • Eleven Labs：AI 语音合成，模仿声音逼真，支持多语言多风格

国内 AI 大模型对比

  • 智谱 GLM：编程+中文理解双强，适合搭配 Claude 做非商业全栈项目，API 便宜
  • 通义千问：生成速度快，API 对接方便，提供免费额度，适合意图识别
  • DeepSeek：完全免费，编程出色，适合技术知识校验
  • Gemini：推理+多模态+超长上下文，适合分析大型项目和联网调研

Claude Code 高级技巧

  • 并行多开：同时运行多个实例，用通知区分需要人工输入的标签页
  • 善用计划模式：复杂任务先打磨计划，方向不对立刻切回重规划
  • 团队维护 CLAUDE.md，用 Git 管理，纠正错误时顺手更新
  • 封装斜杠命令（如 /commit-push-pr），跨项目复用
  • 使用 MCP 扩展功能（Slack、BigQuery、Sentry 等）
  • 完善反馈验证机制：让 AI 自测 UI、自动迭代
  • 提示词要有创意（如”先考考我理解对不对”、”推翻重来给更优雅方案”）
  • 使用子代理（”use subagents”）保持主上下文干净
  • 语音输入描述需求，比打字快三倍且更详细

MCP 服务详解

  • Time MCP：时间相关功能
  • Filecrawl MCP：抓取网页文章内容并深度分析
  • Context7 MCP：获取最新技术文档，解决 AI 训练数据截止问题，免费注册

  • Web to MCP：Chrome 扩展，把网页 UI 发送给 AI 生成代码
  • Chrome DevTools MCP：让 AI 直接控制 Chrome 调试（分析性能、自动填表、测试功能）
  • EdgeOne Pages MCP：腾讯云一键部署，自动打包+上传+部署+CDN 加速，免费
  • GitHub MCP：让 AI 操作 GitHub 仓库（搜索、Issue、PR、分析提交记录）
  • COS MCP：对象存储 MCP，团队协作中共享文件，让 AI 直接读取云端文件

部署 & 上线知识

  • 部署平台：Vercel（最流行）、Netlify（免费额度大、A/B 测试）、EdgeOne Pages（腾讯云）、GitHub Pages（免费无限流量）
  • CDN：Cloudflare 免费 CDN，但 Vercel/Netlify 自带 CDN 无需额外配置
  • 环境变量：本地用 .env，部署时在平台设置中添加，绝不硬编码密钥
  • 域名：国内服务器需备案（1-2 周），国外平台可跳过备案
  • 数据库：推荐 Supabase（PostgreSQL + 认证 + 存储 + 实时同步，免费额度）
  • Git 回退：git log 找版本号 → git reset --hard 版本号；建议每完成一个功能就提交一次

VS Code 扩展推荐

  • GitLens：代码行级历史查看 + AI 生成提交信息/解释变更/生成 Changelog
  • Office Viewer：编辑器内直接浏览 Excel/Word/PDF/SVG 等，万能档案查看器
  • ESLint：前端代码质量检查，AI 项目自带配置
  • Prettier：代码格式化，统一风格，建议开启 Format On Save

  • Error Lens：错误实时内联显示，无需悬停查看

Skills 生态

  • Skills-Click（Vercel）：一行命令安装 Skills，npx skills add owner/repo
  • Find-Skills（Vercel）：交互式搜索 Skills，npx skills find
  • Skill-Creator（Anthropic）：引导创建自定义 Skill
  • Skill-Seeker：自动抓取文档/网页/GitHub 转成 Skills
  • Superpowers：完整 AI 编程技能框架（头脑风暴→计划→执行→TDD→调试→代码审核），适合大型项目
  • Planning with Files：用 Markdown 文件作 AI 外部记忆，解决上下文丢失，适合多步骤/跨对话任务
  • UI UX Pro Max：专业前端设计 Skill，告别千篇一律 AI 风格
  • Vue Skills：尤雨溪团队维护，Vue 3 + Vite + Vitest + Pinia + UnoCSS 最佳实践
  • Supabase Postgres Best Practices：数据库查询优化、索引设计
  • 浏览器自动化：Browser Use、Agent Browser（Vercel）— 自动化测试/爬虫/监控
  • 内容创作：Remotion（代码生成动画视频）、Baoyu-skills（公众号/PPT/小红书配图）、Humanizer（去除 AI 痕迹）
  • Skills 资源平台：Vercel Skills 排行榜 + MCP Market 日报

Spec-kit 规范驱动开发（SDD）流程

核心理念：先写规范文档作为”唯一真理来源”，AI 严格遵守规范生成代码

  1. specify init — 初始化项目
  2. /speckit.constitution — 定义项目”宪法”（基本准则、代码规范）
  3. /speckit.specify — 描述功能需求
  4. /speckit.clarify — 澄清不明确的需求
  5. /speckit.plan — 制定技术方案（生成 Cloud.md、Quickstart.md、Plan.md、Data-model.md 等）
  6. speckit.tasks — 拆解可执行任务列表
  7. /speckit.analyze — 检查规范完整性
  8. speckit.implement — 按任务列表生成代码

OpenSpec 轻量 SDD 框架

比 Spec-kit 更简单，核心流程：变更提案 → 确认后实现 → 归档到规范文档

初始化后生成：

  • OpenSpec-Specs/：主规范文档
  • OpenSpec-Changes/：变更提案
  • OpenSpec-Agents.md：AI 使用指南
  • OpenSpec-Project.md：项目上下文

Typeless 语音输入

  • 识别准确、速度快
  • 支持多种格式（如列表）
  • 实时翻译
  • 可通过描述自动搜索

Vibe Coding 注意事项

  • AI 更适合生成小项目，大体量代码库中 AI Bug 可能导致调试死局
  • 让 AI 生成简单清晰的代码，每次生成后测试
  • 遇到问题及时回滚，不要一条路走到黑
  • 保持手写代码能力，把 AI 当助手而非替代品

视频下载注意事项

  • 站在巨人肩膀上，用 GitHub 开源工具，不自己承担风险
  • 以学习为主，不要侵权
