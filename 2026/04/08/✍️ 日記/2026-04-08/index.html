🌟 Claude Code 创始人神级技巧

  • 并行多开提高效率 在终端中同时运行多个 Claude 将标签页编号 通过系统通知了解哪个需要人工输入 还可以在网页端 Claude Code 上运行多个实例 和本地 Claude 同时进行 充分利用等待时间
  • 善用计划模式，复杂任务一定要让 Claude 先进入计划模式 把心思花在打磨计划上 计划到位了 Claude 基本能一次搞定 一旦感觉方向不对 别硬推 立刻切回计划模式重新来
  • 优先用带思考功能的模型
  • 团队维护 CLAUDE.md 文件，把文件用 git 管理，团队成员共同维护，每次纠正 Claude 的错误，顺手让它把这条经验更新到 Claude.md
  • 封装技能和斜杠命令 重复的事情就封装成技能或斜杠命令，用 Git 管理，跨项目复用 比如搞个 /techdebt 命令每次对话结束跑一下清理重复代码，或者创建 /commit-push-pr 命令一次性完成提交、推送和创建 PR
  • 使用 MCP 工具扩展功能，比如 Slack 搜索和发布内容、运行 Big Query 查询、从 Sentry 获取错误日志等
  • 完善反馈验证机制：别光让 Claude 干活，还要让它知道怎么验证自己的工作，比如让它打开浏览器测试 UI，发现问题后自动迭代，直到功能正常运行
  • 提示词要有创意，比如让 Claude 当考官“先考考我这些改动理解对不对，我过了再提交 PR”，或者说“根据你现在掌握的所有信息，推翻重来，给我个更优雅的方案”
  • 使用子代理：想让 Claude 更快完成任务的时候，在请求后面加一句“use subagents”，把独立的小人物丢给子代理处理，主代理的上下文就能保持干净
  • 选择合适的终端环境：试试事事语音输入，因为说话比打字快三倍描述需求时会写得更详细，AI 理解能更加准确，还能使用 states line 定制状态栏显示上下用量和当前分支

部署托管平台

  • Vercel：最流行的前端部署平台
  • Netlify：和 Vercel 类似，但功能更加全面一些，免费额度更大，​还支持 A/B 测试和分析
  • EdgeOne Pages：腾讯云推出的边缘全栈开发平台，基于腾讯云 EdgeOne 基础设施
  • GitHub Pages：GitHub 提供的免费静态网站托管服务。优点是完全免费，而且无限流量，与 GitHub 无缝集成
  • Cloudflare CDN：如果想让网站访问速度更快，可以使用 Cloudflare 的免费 CDN 服务。
    • CDN（内容分发网络）就是把你的网站内容缓存到全球各地的服务器上，用户访问时会自动选择离它更近的服务器，大大提高加载的速度
    • Cloudflare 的优势包括：
        1. 完全免费
        1. 全球 CDN 覆盖 200 个城市
        1. 自动 HTTPS 证书
        1. DDoS 防护和 Web 防火墙
        1. 免费 DNS 服务
    • 如果你的网站部署在 Vercel 或者 Netlify 上，它们本身就有 CDN 加速，不需要额外配置 Cloudflare。如果是你自己租服务器部署，强烈建议用 Cloudflare 加速
