2026-04 月总结

2026-04-28 5.0k 字 约 17 分钟

2026-04 月总结

范围：根据 2026-04-01 到 2026-04-28 的日记整理。

一句话总结

这个月的主线是：我从“了解 AI 编程工具和后端面试方向”，逐步转向“把 AI 工具用在真实项目、简历包装、面试复盘和 SQL 刷题里”。前半月主要在搭工具、做小项目、准备面试；后半月重心转到后端项目表达、数据开发方向、SQL 练习和个人作品上线规划。

这个月主要做了什么

1. 搭建并理解 AI 编程工作流

  • 系统了解了 Cursor、Claude Code、Gemini CLI、Windsurf、Antigravity、Augment Code 等 AI IDE/CLI 工具。
  • 学习了 MCP、Context7 MCP、Chrome DevTools MCP、GitHub MCP、COS MCP 等工具生态，理解了“让 AI 接入外部能力”的价值。
  • 研究了 Skills 生态，包括 Obsidian Skills、Chops、Superpowers、Planning with Files、Skill-Creator 等。
  • 梳理了 AI 编程的基本原则：
    • 复杂项目先让 AI 分析代码库，而不是直接写代码。
    • 每个独立任务新开对话，减少上下文污染。
    • 使用 .cursorrules、.cursorignore 约束 AI 行为。
    • AI 生成的代码必须人工 Review，关键逻辑需要自己理解和必要时重写。
    • 核心路径要测试、压测、做边界验证。
  • 学习了 Vibe Coding 与 Spec Coding 的区别，进一步认识到：小 Demo 可以快速试错，复杂项目更需要规范、文档和技术契约。
  • 学习了 Claude Code 的使用技巧，包括 /context 监控 Token、/clear 清理上下文、精简 CLAUDE.md、关闭不必要 MCP Server、使用 subagent 和多 Agent 并行。
  • 开始思考多 Agent 开发：主 Agent 负责管理，subagent 并行处理任务，工程师负责判断哪些文件和任务可以并行。

2. 推进了多个个人项目和可展示作品

  • 安装并跑通了 Manim，完成了画圆、方形变圆等动画示例。
  • 用 bolt.new 做了个人主页并上线：https://digital-business-car-t2so.bolt.host
  • 用 bolt.new 做了待办事项应用，迭代了基础功能、UI、优先级、编辑、清空确认和空状态提示，并修复了布局问题。
  • 用 Vibe Coding 做了图片压缩工具并部署到 Vercel：https://compress-photos-orpin.vercel.app
  • 将图床从 GitHub 迁移到阿里云 COS，考虑了隐私和成本。
  • 开发了“万能视频下载器”：
    • 技术栈是 Python、FastAPI、uvicorn。
    • 核心依赖是 yt-dlp。
    • 创建了 GitHub 仓库 free-video-downloader。
    • 修复了 B 站/YouTube 高清视频无声音问题，理解了视频流和音频流分离后需要 ffmpeg 合并。
    • 新增字幕提取功能并测试通过。
    • 已有核心视频下载、视频总结、字幕获取功能。
    • 尝试做字幕时间戳跳转，但点击后无法跳到对应时间点，这个 Bug 还没解决。
  • 开始规划 AI 生成 PPT 前后端分离项目，围绕“尽快上线”调整方案，不盲目上微服务 Gateway，而是倾向 Sa-Token + Redis + BCrypt + RBAC 这类更适合小项目的方案。
  • 后续项目清单逐渐明确：完善个人作品展示网站、购买/配置域名、上线短视频解析平台、上线 AI 生成 PPT 平台、继续完善简历。

3. 集中准备面试和求职

  • 从 4 月 9 日开始认真制定面试准备计划：项目与简历深挖约 1 周，每天梳理 1 个项目和对应必会题。
  • 梳理项目表达模板：业务背景、技术栈、我的角色、负责模块、难点与解决方案、量化结果。
  • 重点准备了三个项目：
    • 泓皓农业：农产品后台管理系统、SPU/SKU、库存扣减、AOP 操作审计、SQL 优化。
    • OJ 在线判题：Spring Boot + Spring Cloud + Docker、工厂模式、策略模式、代理模式、沙箱安全。
    • 云图库：Redis + Caffeine 多级缓存、COS、AI、WebSocket、Disruptor、秒传、分片上传。
  • 为 4 月 13 日上午面试制定了 12-14 小时复习计划，优先级是项目讲述、简历技术深挖、并发/MySQL/集合、JVM/Spring 基础和模拟面试。
  • 面试后做了复盘，更新了《面试手册完整版》：
    • 新增订单状态流转为什么要做幂等控制。
    • 新增 Java 8 Lambda 主要解决什么问题。
    • 新增自定义注解一般用在什么场景。
    • 给库存扣减题补了重点标记。
  • 总结出真实高频方向：系统设计、Redis、多级缓存、一致性、事务与幂等、订单与库存业务规则、分布式/微服务、MySQL、Java 并发、Spring/Spring Boot。
  • 尝试投递了少儿编程、数据开发、数据分析、测试岗位，也开始接受“第一份工作不要太挑”的现实策略。
  • 意识到简历中加入 AI 项目是加分项，也可能帮助转向大模型数据开发、数据分析、产品或测试相关岗位。

4. 复习 Java、Spring 和后端基础

  • 系统整理了 Java 基础：
    • 字符串、数组复制和打印。
    • equals() 与 ==。
    • String 拼接、拆分、源码理解。
    • 类和对象、Object、变量、方法、构造方法、访问权限。
    • 抽象类、接口、内部类、封装、继承、多态。
    • this、super、static、final、不可变对象、重载重写、注解。
    • Java 集合框架、PriorityQueue、ArrayList 和 LinkedList、泛型、迭代器删除问题。
  • 复习了 Spring Cloud 常见组件：Nacos、Sentinel、Seata、RocketMQ、Sidecar、OpenFeign。
  • 持续记录 Spring Security、登录相关内容。
  • 学习 Spring Boot 4.0 相关概念：
    • 基于 Spring Framework 7.0。
    • 对 Java 25 原生支持。
    • HTTP Service Clients。
    • 原生 API 版本管理。
    • JSpecify 空安全体系。
  • 学习了 Redis + Caffeine 多级缓存：
    • 本地缓存 + Redis + DB 的访问链路。
    • 本地缓存减少 Redis 网络 I/O。
    • 难点在数据库一致性、多实例本地缓存一致性和过期策略。
    • 了解了手写缓存、Spring Cache、AOP + 自定义注解三种实现方式。
  • 学习设计模式，重点看了工厂模式，以及创建型、结构型、行为型模式的整体分类。
  • 为测试岗位补充关注点：JUnit、Mockito、Netty/Mina、Activiti、分布式微服务整体流程。

5. 开始明显转向 SQL 和数据开发

  • 4 月 24 日开始看数据开发成长路线，明确了入门阶段要做的事：每天写 SQL、接简单取数需求、整理笔记、找人请教。
  • 记录了数据开发和 AI 结合的思路：PRD 给 AI 读、提示词优化、模型温度、不要给大模型 drop table 权限。
  • 4 月 27 日集中整理和练习了 49 道 SQL 题，覆盖：
    • 建表、插入、索引、强制索引、触发器、视图、外键。
    • 字符串函数、group_concat、更新、删除、替换。
    • group by、not in、join、left join、子查询、exists、case。
    • 平均值、累计和、排名、分页、订单分析、商品交易、简历投递分析等业务题。
  • 4 月 28 日继续补了数据分析常用 SQL：
    • 窗口函数：row_number、rank、dense_rank、percent_rank、lag、lead 等。
    • 日活、月活、新用户、次留、7 日留存、15 日留存。
    • 最大连续登录天数。
    • 中位数、去掉最大最小后的平均值、分组累计求和。
  • 开始收集系统设计和 Vue 练习资源，并产生了“做一个刷题网站”的想法：导入题库、AI 分析、刷题、错题回顾、知识点整理。

6. 读了一些方法论和人生判断

  • 读了“善念”和“善果”的故事，得到的结论是：善意本身不够，关键是有没有能力、远见和后续安排把善意变成好结果。
  • 读到“不与命斗、不与法斗、不与理斗、不与势斗”，理解成不是不作为，而是识别边界，把精力放到自己能影响的地方。
  • 对 AI 编程的态度更清醒：AI 很强，但不能替代手写能力和工程判断。它适合做第一稿、解释代码、生成测试、辅助 Review、起草文档和缩小排查范围。
  • 对求职的判断也更现实：第一份工作不必过度挑剔，要先进入真实环境，积累项目经验和表达素材。

时间线回顾

  • 4 月 1 日到 4 月 7 日：安装 Manim，补 Java 基础，了解 Spring Cloud 和 AI 编程原则。
  • 4 月 8 日到 4 月 10 日：高强度了解 AI 工具链、MCP、Skills、部署平台；做个人主页、Todo、图片压缩工具；开始视频下载器。
  • 4 月 11 日到 4 月 13 日：围绕 4 月 13 日面试进行项目讲述和技术高频点突击。
  • 4 月 15 日到 4 月 22 日：学习多 Agent、Vim/命令行工作流、Spring Security、Spring Boot 4.0、HTTP Service Clients、JSpecify；规划 AI PPT 项目。
  • 4 月 19 日和 4 月 24 日：做面试复盘，更新面试手册，明确高频考点和求职方向，开始关注数据开发路线。
  • 4 月 27 日到 4 月 28 日：集中刷 SQL 和整理数据分析指标 SQL，重心明显转向数据开发和面试实战题。
